# SunoLyricist 專案探索發現

## 專案現狀分析

### 已完成功能 (優秀)
1. **AI 創作三模式**：手動 / AI 優化 / AI 全自動
2. **完整的 Style Prompt 建構器**：風格、樂器、人聲、節奏
3. **歌曲結構編輯器**：拖放排序、預設模板
4. **迭代優化功能**：可針對歌詞或 Style 分別優化
5. **版本歷史系統**：LocalStorage 持久化、匯出/匯入
6. **雙認證模式**：OAuth (claude login) + 直接 API Key
7. **Metatag 參考文檔**：完整的標籤說明

### 技術架構
- **前端**：純 HTML/CSS/JavaScript（~2000 行 JS，~1650 行 CSS）
- **後端**：FastAPI + Claude Code SDK（~160 行 Python）
- **無框架依賴**：輕量、易維護

## 涌現機會發現

### 機會 1: 批量生成與比較
**現象**：用戶可能想一次生成多個版本比較
**潛在應用**：
- 一鍵生成 3 個不同風格的版本
- 並排比較、混合選擇最佳段落
- 參考成功生成的模式

### 機會 2: Style Prompt 模板庫
**現象**：每次都要重新設定 Style 很繁瑣
**潛在應用**：
- 預設風格模板（抒情、嘻哈、電子等）
- 用戶可保存自己的風格組合
- 社群分享熱門 Style 組合

### 機會 3: 歌詞結構分析
**現象**：生成的歌詞可能結構不佳
**潛在應用**：
- 分析歌詞押韻模式
- 檢測段落平衡度
- 建議結構優化

### 機會 4: Suno API 直接整合（長期）
**現象**：目前只生成歌詞，需手動貼到 Suno
**潛在應用**：
- 直接調用 Suno API 生成音樂
- 在應用內預覽生成結果
- 一站式創作流程

### 機會 5: 多語言歌詞互譯
**現象**：已支援多語言選項
**潛在應用**：
- 將中文歌詞翻譯成英文版本
- 保持押韻和節奏感的翻譯
- 多語言版本對照

## 代碼品質觀察

### 優點
- CSS 變數系統完善，易於主題切換
- 事件綁定集中管理
- 響應式設計完整

### 可改進
- `app.js` 單檔案 ~2000 行，可考慮模組化
- 無 TypeScript，大型專案可能難以維護
- 未使用構建工具，無法享受 tree-shaking
